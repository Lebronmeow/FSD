Introduction to MongoDB

storage What is MongoDB?

  • NoSQL database
  • Stores data in JSON-like documents
  • No tables and rows
  • Organizes data in collections
  • Handles unstructured/semi-structured data

architecture Key Features

  • Flexible schema
  • Horizontal scaling
  • High availability
  • Rich query language
  • Document-oriented storage

Query Operators

compare_arrows Comparison Operators

  • $gt - Greater than
  • $gte - Greater than or equal
  • $lt - Less than
  • $lte - Less than or equal
  • $eq - Equal
  • $ne - Not equal

logic Logical Operators

  • $and - Logical AND
  • $or - Logical OR
  • $not - Logical NOT
  • $nor - Logical NOR
// Example: Find movies with year > 2000 AND rating >= 8 { "year": { "$gt": 2000 }, "rating": { "$gte": 8 } }

Working with Query Results

settings Query Result Operations

  • Sorting: Arrange documents in order
  • Limiting: Restrict number of results
  • Projection: Select specific fields
  • _id Field: Unique identifier

Key Points

  • Queries are written in JSON format
  • Filtering based on specific conditions
  • Query operators enable advanced queries
  • Each document has a unique _id field
14 / 19

Sorting, Limiting, Projection

Operation Description Example
Ascending Sort Sort data in increasing order { "year": 1 }
Descending Sort Sort data in decreasing order { "year": -1 }
Limit Restrict number of documents Limit = 2
Include Fields Show selected fields only { "title": 1, "year": 1 }
Exclude _id Remove _id from output { "_id": 0 }

Understanding _id Field

fingerprint Properties

  • Unique: Each document has unique identifier
  • Auto-generated: Created by MongoDB
  • Immutable: Cannot be modified

code Example

{ "_id": "573a1390f29313caabcd42e8", "title": "The Great Train Robbery", "year": 1903 }

Important Note

By default, _id is included in query results unless explicitly excluded using projection: { "_id": 0 }
